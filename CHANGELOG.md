# Change Log

## RSL-plus 1.1.4

- Добавлен настоящий `FastDocumentSnapshot`: один lexer-проход сразу формирует Folding, Outline, Import и индекс идентификаторов; полный parser переиспользует готовый lexer.
- Folding и Outline больше не ожидают построения полного `CBase`.
- `RslScopeResolver.resolveAt()` кэшируется по версии модуля, Import-closure и позиции токена.
- Semantic Tokens Range разрешает только токены запрошенных строк, не строя полный набор токенов файла.
- `ONERROR` верхнего уровня перенесён в исходную grammar и structural-анализ; удалён post-filter ложных диагностик.
- References обрабатывает точные parser-кандидаты с CPU-бюджетом 8 мс между файлами.
- ReferenceIndex выделен в самостоятельный сервис и хранит отсортированные уникальные 32-bit hash вместо фиксированного Bloom-фильтра.
- Инвалидация ReferenceIndex стала адресной: изменение одного файла не очищает индекс остальных файлов.
- Кандидаты References ограничиваются модулем объявления и его транзитивными Import-dependents; при непрогретом графе используется безопасный workspace fallback.
- ReferenceIndex сохраняется в storage расширения и восстанавливается по `mtime + size`, не удерживая исходные тексты или AST.
- Архитектурные performance-тесты объединены в три сценария: pipeline, semantic resolution и reference index; семь дублирующих тестовых файлов выведены из набора.
- Исправлена семантика `ONERROR`: обработчик разрешён на верхнем уровне исполняемого макрофайла и не обязан завершаться отдельным `END`.
- Find All References для локальных переменных и параметров больше не обходит workspace — поиск ограничивается текущим Macro/Method.
- Внешние файлы для References читаются пакетами по 16 с ограниченным параллелизмом вместо последовательного обхода.
- Fast snapshot открытого файла строится сразу; полный parser запускается после 25-мс приоритетного окна, изменения текста объединяются короткой задержкой.
- Один parser/lexer snapshot используется для symbol tree, folding, Outline и навигации — повторное лексирование при открытии устранено.
- Problems разделены на локальную и Import-зависимую фазы: локальные ошибки появляются быстро, workspace-проверки обновляют результат позже.
- Workspace-диагностика получила максимальный срок ожидания 1,8 секунды и больше не переносится бесконечно при последовательной загрузке Import.
- Ключ локальной диагностики больше не зависит от глобальной ревизии workspace; workspace-фаза зависит только от текущего Import-графа.
- External summary строится отдельным однопроходным scanner-ом без statement/expression AST и хранит готовые позиции внешних символов.
- Переход к объекту закрытого импортированного файла использует сохранённый Range и не перечитывает файл только ради positionAt.
- Три отдельных Import LRU объединены в один ленивый `ImportContext` на 8 активных документов.
- Semantic Tokens поддерживают delta и range-запросы; неизменённая версия возвращает пустой delta.
- Find All References больше не перестраивает глобальные symbol/import индексы для временно разобранных файлов и не создаёт lowercase-копию всего файла.
- Инвентаризация `**/*.mac` запускается через 500 мс после готовности активного документа и исключает archive/backup/history/build-каталоги.
- Обновление счётчика модулей в status bar ограничено частотой, чтобы не создавать поток IPC-сообщений.
- Language server разделён на сервисы анализа документов, диагностик, настроек, загрузки workspace и LSP-provider-ов.
- Файлы `server/src` сгруппированы по каталогам `analysis`, `diagnostics`, `features`, `indexing`, `services` и `core`.
- Устранено смешивание resource-настроек между открытыми файлами.
- По умолчанию отключена полная фоновая индексация workspace: загружается только транзитивная цепочка `Import` открытых документов.
- При закрытии редактора полная модель заменяется компактным external summary без исходного текста, lexer-токенов, AST и локальных объявлений Macro.
- Удалён отдельный `externalModuleCache` из Definition Provider; владельцем загруженных модулей остаётся `WorkspaceIndex`.
- Import-кэши ограничены LRU-размером и инвалидируются только для изменённого модуля и его зависимых файлов.
- Find All References читает неизвестные файлы по запросу, применяет дешёвый текстовый prefilter и освобождает временный parser result после проверки файла.
- Диагностики рассчитываются только для активного RSL-документа, повторные вычисления одной версии пропускаются, лимит Problems передаётся непосредственно правилам.
- Изменения закрытых `.mac`, не входящих в активный Import-граф, больше не приводят к их автоматической загрузке.
- Неоднозначные `Import` не разрешаются случайным выбором файла и выводятся в Problems.
- Диагностические правила подключаются через `RslDiagnosticEngine`, Quick Fix — через `RslQuickFixRegistry`.
- Стабилизирована видимость Problems: фоновые файлы не перехватывают активный документ.
- Полностью обновлён README.

## RSL-plus 1.1.3

- Добавлен tolerant syntax parser для основных конструкций RSL.
- Пропущенные `;`, `,`, `)` и `END` фиксируются при построении дерева.
- Добавлено восстановление parser после синтаксических ошибок.
- Соседние строковые литералы распознаются как допустимая неявная конкатенация: вместо ложной ошибки о `;` показывается предупреждение с Quick Fix для вставки `+`.
- Диагностика открытых файлов в **Problems**: переменные, известные Import, DEBUGBREAK, структура блоков, использование до объявления и неоднозначные ссылки.
- Problems очищается при закрытии файла; проверки можно включать и выключать в настройках.
- Добавлены Quick Fixes, Find All References и семантическая подсветка символов.
- Улучшены переходы по Import и устойчивость больших SQL-блоков.
- Уменьшен VSIX: корневые devDependencies исключены из упаковки.
