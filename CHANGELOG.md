# Change Log

## RSL-plus 1.1.4

- Language server разделён на отдельные сервисы анализа, диагностики, навигации и загрузки workspace.
- Ускорены открытие и обработка RSL-файлов: lexer и parser переиспользуют общие результаты, повторные изменения объединяются.
- Folding и Outline формируются без ожидания полного разбора документа.
- Диагностика разделена на локальную и Import-зависимую фазы, благодаря чему основные Problems появляются быстрее.
- Улучшена работа с Import: загружается только цепочка зависимостей открытых файлов, а закрытые модули хранятся в компактном виде.
- Оптимизированы Semantic Tokens, переход к определению и разрешение символов.
- Find All References получил отдельный индекс, локальный поиск для переменных и параметров и пакетную обработку файлов.
- Исправлена поддержка верхнеуровневого ONERROR.
- Исправлено распознавание наследования в конструкциях CLASS (BaseClass) ChildClass.
- Устранены ложные неоднозначные ссылки, случайное разрешение неоднозначных Import и перехват Problems фоновыми файлами.
- Обновлены и объединены архитектурные и регрессионные тесты.
- Полностью обновлён README.

## RSL-plus 1.1.3

- Добавлен tolerant syntax parser для основных конструкций RSL.
- Пропущенные `;`, `,`, `)` и `END` фиксируются при построении дерева.
- Добавлено восстановление parser после синтаксических ошибок.
- Соседние строковые литералы распознаются как допустимая неявная конкатенация: вместо ложной ошибки о `;` показывается предупреждение с Quick Fix для вставки `+`.
- Диагностика открытых файлов в **Problems**: переменные, известные Import, DEBUGBREAK, структура блоков, использование до объявления и неоднозначные ссылки.
- Problems очищается при закрытии файла; проверки можно включать и выключать в настройках.
- Добавлены Quick Fixes, Find All References и семантическая подсветка символов.
- Улучшены переходы по Import и устойчивость больших SQL-блоков.
- Уменьшен VSIX: корневые devDependencies исключены из упаковки.
