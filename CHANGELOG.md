# Change Log

## RSL-plus 1.1.3 — диагностика Problems

Версия добавляет серверную диагностику RSL. Сообщения выводятся в панели VS Code **Problems** и обновляются после изменения документа или импортированного файла.

### Неиспользуемые объявления

Проверяются:
- локальные переменные;
- локальные константы;
- параметры Macro и методов;
- глобальные объявления с модификатором `Private`.

Пример:
```rsl
Macro Test(usedParam, unusedParam)
    Var usedValue, unusedValue;
    usedValue = usedParam;
End;
```

В Problems появятся:
```text
Параметр unusedParam объявлен, но не используется
Переменная unusedValue объявлена, но не используется
```

Публичные глобальные переменные и константы не проверяются: они могут использоваться внешней средой или импортирующими файлами.
Старый клиентский regex-декоратор удалён. Анализ выполняется один раз в language server по lexer/AST и не сканирует большой файл повторно в Extension Host.

### Неиспользуемые Import

Для каждого прямого Import проверяется, используется ли в текущем файле хотя бы один публичный объект импортированного модуля или его транзитивных Import:

- глобальная переменная;
- константа;
- Macro;
- Class.

Проверка использует scope resolver, а не простое совпадение текста. Локальное объявление с таким же именем не считается использованием импортированного объекта.

Строковый вызов учитывается:

```rsl
ExecMacro("ImportedMacro");
ExecMacro2("ImportedFunction");
```

Сообщение имеет уровень Hint и сформулировано как `возможно, не используется`, потому что некоторые модули могут импортироваться ради обязательного исполняемого блока.

Модуль без публичных объявлений не помечается как неиспользуемый — он может содержать только side effects.

### Структура кода

Проверяются:

- лишний `END`;
- отсутствующий `END` у `MACRO`, `CLASS`, `IF`, `FOR`, `WHILE`;
- `ELIF` и `ELSE` без соответствующего `IF`;
- `ONERROR` вне `MACRO`;
- лишние и незакрытые круглые скобки `()`;
- лишние и незакрытые фигурные скобки `{}`;
- незакрытая строка;
- незакрытый многострочный комментарий;
- незакрытый блок `[ ... ]`.

Содержимое строк, комментариев и SQL/PLSQL-блоков `[ ... ]` в баланс RSL-блоков и скобок не входит.

Поскольку RSL использует универсальный `END`, при пропуске внутреннего `END` последний `END` может формально закрыть внутренний блок. В таком случае диагностика покажет незакрытым внешний блок — это ожидаемое ограничение синтаксиса без типизированных `END IF`/`END LOOP`.

### Дополнительные проверки

Также добавлены:

- импортируемый файл не найден в workspace;
- файл импортирует сам себя;
- один модуль импортирован повторно;
- имя повторно объявлено в одной области видимости;
- прежнее информационное сообщение об устаревших `RECORD` и `ARRAY`.


## 1.1.0 — lexer, scopes and workspace index

Это overlay для актуального репозитория `aschumakov/RSL-plus`.
Содержимое архива нужно распаковать в корень репозитория с заменой файлов.
Неизменённые файлы проекта в архив не включены.

### Реализовано

1. Безопасное форматирование:
   - не изменяет строки в `"..."` и `'...'`;
   - не анализирует содержимое `[ ... ]` как RSL;
   - сохраняет BOM, CRLF/LF и финальный перевод строки;
   - возвращает пустой набор TextEdit, когда изменений нет.
2. Разрешение имён по областям видимости:
   - локальный Macro/Method;
   - текущий Class;
   - модуль;
   - импортированные модули;
   - переходы к методам учитывают объект слева от точки.
3. Watcher для `**/*.mac` и инвалидизация кэшей при создании,
   изменении, удалении и переименовании файлов.
4. Completion и Hover перед ответом получают дерево текущей версии документа.
5. Общий `npm test`, тесты lexer/formatter/folding/definition/parser/scope,
   GitHub Actions CI и публикация только по тегу или вручную.
6. Общий lexer используется token cache, форматтером, folding,
   ExecMacro-навигацией, диагностикой и индексом импортов.
7. Ленивый индекс всех `.mac` проекта, индекс загруженных символов,
   прямой и обратный граф Import.
