# Change Log

## RSL-plus 1.1.5

### IntelliSense и навигация
* Добавлен Signature Help: при вводе вызова показываются параметры Macro/Method и активный аргумент, включая вложенные вызовы.
* Добавлен Call Hierarchy для просмотра входящих и исходящих вызовов Macro/Method.
* Completion предлагает доступные символы из известных модулей с автоматической вставкой `Import`.
* Quick Fix для неизвестного идентификатора выполняет точный поиск экспортирующего модуля по workspace и добавляет отсутствующий `Import`.
* Передача локальной переменной по ссылке (`@variable`) учитывается как использование и поддерживается навигацией по символу.
* Общесистемные спецпеременные RS-Bank, включая `{oper}`, распознаются без ложной ошибки о неоднозначном Import; их список и типы дополнены по документации.

### Редактор
* Добавлено форматирование выделенного диапазона. Форматтер учитывает окружающую структуру, но создаёт изменения только для выбранных строк.
* Форматтер расставляет пробелы вокруг `=` в присваиваниях и после запятых в списках аргументов и объявлений.
* Добавлена команда сворачивания всех Macro в открытом файле.

### Производительность
* Auto Import не запускает обход всего workspace во время обычного Completion. Полный поиск выполняется только при явном вызове Quick Fix и кэшируется.

## RSL-plus 1.1.4

### Производительность
* Ускорено открытие и повторное редактирование RSL-файлов: lexer и parser переиспользуют общие результаты, а частые изменения текста объединяются.
* Folding и Outline появляются без ожидания полного синтаксического разбора и кэшируются для текущей версии файла.
* Диагностика разделена на быстрые локальные проверки и последующий анализ Import-зависимостей.
* Загружается только цепочка Import открытых документов, а закрытые модули хранятся в компактном виде, что снижает потребление памяти на больших проектах.

### Навигация и работа с кодом
* Hover теперь показывает сигнатуру, параметры, тип результата, видимость, базовый класс и документацию символа.
* Добавлена подсветка использований выбранного символа с различием чтения и записи.
* Добавлены переходы к началу и концу текущего блока, а также улучшено расширение выделения для вложенных конструкций.
* Ускорены Go to Definition и Find All References: локальные символы ищутся только внутри своего блока, а для поиска по проекту используется отдельный индекс.

### Parser, Problems и Import
* Добавлено различение SQL-блоков и выходных форм внутри `[ ... ]`.
* Реализован разбор аргументов и спецификаторов форматирования выходных форм с семантической подсветкой и пояснениями в Hover.
* Добавлена диагностика циклических Import с отображением цепочки зависимостей.
* Неоднозначные Import больше не разрешаются случайным выбором файла; устранены ложные сообщения о неоднозначных ссылках и стандартных типах RSL.
* Исправлена поддержка верхнеуровневого `ONERROR`, конструкций `CLASS (BaseClass) ChildClass` и стандартных типов RSL.
* Problems стабильно показывает ошибки активного документа и не заменяет их результатами фонового анализа других файлов.

### Форматирование
* Исправлены отступы внутри верхнеуровневого `ONERROR`.
* Добавлено выравнивание последовательных присваиваний по знаку `=`.

### Надёжность
* Переработана внутренняя организация language server и расширен набор автоматических тестов для parser, диагностики, Import, навигации, форматирования и производительности.

## RSL-plus 1.1.3

- Добавлен tolerant syntax parser для основных конструкций RSL.
- Пропущенные `;`, `,`, `)` и `END` фиксируются при построении дерева.
- Добавлено восстановление parser после синтаксических ошибок.
- Соседние строковые литералы распознаются как допустимая неявная конкатенация: вместо ложной ошибки о `;` показывается предупреждение с Quick Fix для вставки `+`.
- Диагностика открытых файлов в **Problems**: переменные, известные Import, DEBUGBREAK, структура блоков, использование до объявления и неоднозначные ссылки.
- Problems очищается при закрытии файла; проверки можно включать и выключать в настройках.
- Добавлены Quick Fixes, Find All References и семантическая подсветка символов.
- Улучшены переходы по Import и устойчивость больших SQL-блоков.
- Уменьшен VSIX: корневые devDependencies исключены из упаковки.
