# RSL-plus 1.1.0 — lexer, scopes and workspace index

Это overlay для актуального репозитория `aschumakov/RSL-plus`.
Содержимое архива нужно распаковать в корень репозитория с заменой файлов.
Неизменённые файлы проекта в архив не включены.

## Реализовано

1. Безопасное форматирование:
   - не изменяет строки в `"..."` и `'...'`;
   - не анализирует содержимое `[ ... ]` как RSL;
   - сохраняет BOM, CRLF/LF и финальный перевод строки;
   - возвращает пустой набор TextEdit, когда изменений нет.
2. Разрешение имён по областям видимости:
   - локальный Macro/Method;
   - текущий Class;
   - модуль;
   - импортированные модули;
   - переходы к методам учитывают объект слева от точки.
3. Watcher для `**/*.mac` и инвалидизация кэшей при создании,
   изменении, удалении и переименовании файлов.
4. Completion и Hover перед ответом получают дерево текущей версии документа.
5. Общий `npm test`, тесты lexer/formatter/folding/definition/parser/scope,
   GitHub Actions CI и публикация только по тегу или вручную.
6. Общий lexer используется token cache, форматтером, folding,
   ExecMacro-навигацией, диагностикой и индексом импортов.
7. Ленивый индекс всех `.mac` проекта, индекс загруженных символов,
   прямой и обратный граф Import.

## Применение в PowerShell

Из корня репозитория:

```powershell
# Распаковать overlay в корень с заменой файлов, затем:
npm.cmd install --package-lock-only --ignore-scripts
npm.cmd ci
npm.cmd test
vsce.cmd package
```

Ожидаемое имя пакета:

```text
rsl-plus-1.1.0.vsix
```

Установка для проверки:

```powershell
code.cmd --install-extension .\rsl-plus-1.1.0.vsix --force
```

## Важное изменение публикации

`.github/workflows/deploy.yml` больше не публикует расширение после каждого
push в `master`. Публикация запускается:

- вручную через `workflow_dispatch`;
- при push тега вида `v1.1.0`.

Перед публикацией выполняются компиляция и все тесты.

## Проведённые проверки

В подготовленной среде выполнены:

- typecheck всех изменённых server-файлов;
- typecheck `client/src/extension.ts`;
- 39 unit/regression checks;
- 100 000 поисков token в parser test — около 8–10 мс.

Проверка выполнялась с типовыми заглушками API VS Code/LSP, потому что
контейнер не имел сетевого доступа для установки npm-зависимостей.
Поэтому финальный `npm.cmd test` и сборку VSIX необходимо выполнить
в самом репозитории с его реальными `node_modules`.
